Analisis Inicial de Funciones y Tareas

   De los requisitos iniciales planteados, se extraen una serie de funciones básicas a implementar, además de una serie de tareas adicionales a desarrollar dentro del sistema.

Roles / Perfiles:
  • Usuario Invitado (guest): Cualquier usuario que navega por las partes públicas del portal, sin necesidad de estar registrado o validado en la aplicación.
  • Usuario Registrado (normal): Pensado para los usuarios que quieren utilizar la web a la hora de buscar, consultar / visualizar las ofertas de su interés, gestionar su perfil, gestionar sus avisos, notificaciones y mensajes con moderadores o administradores, gestionar sus propios ofertas, las ofertas de las que es seguidor,las preferencias de etiquetas, las valoraciones y comentarios creados, etc.
  • Usuario Moderador (moderador): Pensado para los usuarios que además deseen colaborar con la web a la hora de moderar y mantener los datos que se manejan, como son las ofertas, valoraciones y comentarios.
  • Usuario Anunciante / Patrocinador (patrocinador): Pensado para los usuarios que deseen poner sus anuncios de forma destacada o "antes" que otros para que lleguen más veces a la vista de los usuarios que visitan la página.Deberán solicitar su activación como anunciantes mediante un aviso/incidencia especial, para que los administradores le den el permiso en la aplicación.
  • Administrador del Portal (admin): Pensado para aquellos usuarios que realizan las tareas de mantenimiento de los datos del sistema, pueden gestionar cualquier cosa de la parte privada como usuarios, ofertas y datos asociados, valoraciones/comentarios, etiquetas, patrcinadores, configuración, hacer copias de seguridad o restaurarlas.
  • Administrador del Sistema (sysadmin): Pensado para los programadores de la aplicación y/o administradores generales del sistema donde se instale, tendrá acceso a todas las funciones de la aplicación sin restricciones, y opcionalmente deberá poder activar el rol que desee para trabajar como un usuario de ese tipo.
